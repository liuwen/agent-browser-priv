# Patchright Backend

The `agent-browser-priv` package installs the `agent-browser` command and uses
Patchright as the default local Chrome-compatible backend. This is intended for
daily development and remote or ephemeral environments where Patchright-managed
browser artifacts are useful.

Install the backend once:

```bash
agent-browser install
```

Then run normal browser commands:

```bash
agent-browser --headed open https://example.com
agent-browser --profile ~/.agent-browser/profiles/dev open https://example.com
```

Use the built-in Chrome CDP launcher when a site behaves better on that lane:

```bash
agent-browser install chrome
agent-browser --backend chrome open https://example.com
```

Patchright launches a Chromium-compatible browser and exposes CDP on localhost.
The profile is persistent when `--profile` is provided; otherwise agent-browser
uses a temporary profile and removes it when the session closes. The normal
agent-browser commands still run through the Rust CLI.

This backend requires Node.js at runtime because a small host process launches
Patchright before the Rust daemon attaches over CDP. `agent-browser install`
uses a pinned Patchright package lock and installs Patchright's browser
artifacts.

Patchright updates are not picked up automatically from npm. They are pinned in
the `agent-browser` release so browser behavior stays reproducible. Run
`agent-browser doctor` to see the installed Patchright version, the release pin,
and npm latest when network checks are enabled. After upgrading `agent-browser`,
run `agent-browser install` again to refresh the backend to the new pin.

The backend honors proxy, proxy bypass, user agent, ignore-HTTPS-errors,
download path, color scheme, and custom launch args. Remote-debugging address
and port args are reserved and forced to localhost.

This backend is for authorized local development and debugging. It does not
solve CAPTCHA, Turnstile, or other human verification pages. If a challenge
remains, preserve the page state and artifacts for human handoff.
